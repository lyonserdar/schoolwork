Final Project Documentation

This game is optimized for Google Chrome.

This is a sudoku game that has the following features:

Game grid that uses CSS and is generated by JavaScript

The sudoku board and number selection board html is generated by JavaScript. The functions createGameBoard and createGameSelectOptions creates the table elements and adds a css styling to it. The script uses document.createElement method to generate the html.

User input in the form of text, select options, and/or buttons

The number selection board lets users select which number they would like to place on the board. There are several buttons in the game like starting a new game or loading the last played puzzle.

User input in the form of mouse clicks

User uses mouse to play the game. The number selection and placing the selected number to the board listens for mouse click event.

Dynamic modification of HTML using innerHTML or DOM functions

Once the html loaded the game will create the sudoku board and number selection pad. Once the difficulty selected and new game button clicked, puzzle will be loaded to the sudoku board. When you click on a board cell, it will display the number selected. If it is wrong it will indicate the error by changing the style temporarily.

Use of XMLHttpRequest to load JSON or XML game data that is displayed on the grid page

All the puzzles for this game loaded by XMLHttpRequest. The puzzles are stored in a json file called puzzles.json in data folder. You can see this in fetchPuzzles function.

Incorporates HTML 5 audio tags, video tags, canvas, or local storage

When you select a difficulty and start a new game a random puzzle with that difficulty will be loaded. The game stores this puzzle to local storage. If you start a game using start new game with last puzzle. The script will fetch the last played puzzle and start a new game with it.

Game logic code

Sudoku puzzle's given numbers are styled bold.

Given numbers also cannot be changed.

There is a feedback for incorrect numbers. (Animation and chime)

There are three difficulties (also additional debug difficulty for testing) The game will select a random puzzle for each difficulty.

Puzzles are loaded using Ajax

Once a new game started the timer will show how long the player working on the puzzle. Once the game ends the timer will stop.

Extra Credit Include a second HTML5 component.

I used audio for the extra credit. When user selects a wrong number for the sudoku cell, wrong selection audio will play. Also there is a different audio for correct numbers.
